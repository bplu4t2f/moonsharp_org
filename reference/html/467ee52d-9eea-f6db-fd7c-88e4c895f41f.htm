InteropAccessMode Enumeration
Enumerations of the possible strategies to marshal CLR objects to MoonSharp userdata and functions when using automatic descriptors. Note that these are "hints" and MoonSharp is free to ignore the access mode specified (if different from HideMembers) and downgrade the access mode to "Reflection". This particularly happens when running on AOT platforms like iOS. See also : CallbackFunction and UserData .

Members
  Member nameValueDescription
Reflection0 Optimization is not performed and reflection is used everytime to access members. This is the slowest approach but saves a lot of memory if members are seldomly used.
LazyOptimized1 Optimization is done on the fly the first time a member is accessed.
Preoptimized2 Optimization is done at registration time
BackgroundOptimized3 Optimization is done in a background thread which starts at registration time. If a member is accessed before optimization is completed, reflection is used.
HideMembers4 No optimization is done, and members are not accessible at all.
Default5 Use the default access mode
