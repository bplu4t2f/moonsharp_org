MoonSharp.Interpreter.Platforms Namespace
Namespace containing classes used to customize how the interfacing with the operative system happens for the 'os' and 'io' modules, and for future modules/classes which may require OS access. See IPlatformAccessor and Platform .
Classes
  ClassDescription
LimitedPlatformAccessor
A class implementing all the bits needed to have a minimal support of a platform. This does not support the 'io'/'file' modules and has partial support of the 'os' module.
PlatformAccessorBase
An abstract class which offers basic services on top of IPlatformAccessor to provide easier implementation of platforms.
PlatformAutoDetector
A static class offering properties for autodetection of system/platform details
StandardPlatformAccessor
Class providing the IPlatformAccessor interface for standard full-feaured implementations.
Interfaces
  InterfaceDescription
IPlatformAccessor
Interface to abstract all accesses made to the underlying platform (OS, framework) by the scripting engine. Can be used both to support "non-standard" platforms (i.e. non-posix, non-windows) and/or to sandbox the behaviour of the scripting engine. It's recommended that no class implement IPlatformAccessor directly, and rather extend PlatformAccessorBase.
